# Survey Mode

연구 동향을 조사하고 구조화된 보고서로 정리하는 단계다. Research의 확장이다.

## Flow
1. 조사 범위 정의 — 주제, 기간, 포함/제외 기준
2. 분류 체계 수립 — 어떤 축으로 정리할지 (기술별? 시간순? 문제별?)
3. 탐색 — subagent 병렬 실행 (영역별 분담)
4. 수집 결과를 분류 체계에 매핑
5. 비교표/타임라인/트렌드 맵 작성
6. Gap 분석 — 아직 해결되지 않은 문제, 연구가 부족한 영역
7. 보고서 작성 → 파일로 저장

## 보고서 구조 (기본 템플릿)
```
# [주제] 연구 동향 (기간)

## 1. 개요
- 조사 범위와 방법

## 2. 핵심 트렌드
- 트렌드별 설명 + 대표 연구/제품

## 3. 주요 모델/기술 비교
- 비교표 (성능, 접근법, 장단점)

## 4. Open Problems
- 아직 해결되지 않은 문제

## 5. 전망
- 향후 방향 예측 ([INFERENCE] 라벨)

## 참고문헌
- [CITED] 출처 목록 (URL + 날짜)
```

## Rules
- Research 모드의 모든 규칙을 따른다 (증거 라벨링, 교차 확인, subagent 위임)
- 비교표는 최소 1개 포함 — 텍스트보다 표가 동향 파악에 효과적
- 트렌드는 3-5개로 압축 — 너무 많으면 "동향"이 아니라 "나열"
- 전망은 반드시 [INFERENCE] 라벨 — 사실과 추측을 구분
