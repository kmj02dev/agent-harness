# Harness Development Mode

에이전트 행동을 제어하는 텍스트(CLAUDE.md, 모드 파일, 도메인 파일)를 작성·개선하는 단계다. 대상은 코드가 아니라 행동 규칙이다.

## Flow (GAN 루프)
1. 행동 문제 정의 — 에이전트가 어떤 실수를 하는가? 구체적 사례 수집
2. 최소 변경으로 규칙/flow 작성
3. 적용 → 테스트 (실제 시나리오 or subagent 시뮬레이션)
4. 행동 변화 분석 — 변했는가? 의도대로인가? 부작용은?
5. 수정 → 재테스트, 수렴까지 반복

## Rules
- **모든 줄이 행동을 바꿔야 한다.** 바꾸지 않는 줄은 삭제. 토큰은 희소 자원이다.
- **한 번에 하나의 행동 변화에 집중.** 규칙 과다 투입은 교훈 과다 투입과 같다 — 아무것도 정착하지 않는다.
- **기존 패턴을 따른다.** 모드 파일: purpose → Flow → Rules. 도메인 파일: base.md + [mode].md 2계층.
- **검증 없이 "완료" 선언 금지.** 검증 = 실제 행동 변화 확인. 텍스트를 작성한 것은 검증이 아니다.
- **새 모드/도메인 추가 시 정당성 판단.** "이것이 없으면 에이전트가 어떤 실수를 하는가?" 실수를 막지 않으면 불필요.
- **변경 시 관련 파일 동기화 확인.** 모드 파일 수정 후 CLAUDE.md 모드 목록, WORKLOG, TODO와 불일치가 없는지 점검.
