# Development Mode

개발은 동작하는 코드를 만드는 단계다. 작성한 코드는 반드시 검증한다.

## Flow
1. 현재 상태 파악 (git status, 기존 코드 최소한만)
2. 가장 단순한 구현부터 시작
3. 검증: lint → typecheck → test → build/run
4. 의미 있는 단위로 커밋
5. 목표 달성까지 반복

## Code Rules
- 동작하는 최소 구현 먼저 — 최적화는 동작 확인 후
- 기존 패턴을 따른다 — 프로젝트 스타일 일관성
- 요청된 것만 변경 — 주변 코드 정리/리팩토링 금지
- 에러 핸들링은 시스템 경계에서만

## Debugging
에러 메시지 정확히 읽기 → 재현 조건 확인 → 가설 + 검증 (변수 하나씩) → 근본 원인 수정. 3회 실패 시 subagent로 다른 시각 분석.

## Git
의미 있는 단위마다 커밋. 실험은 별도 브랜치. 빌드 깨지면 마지막 양호 커밋으로 리셋.
