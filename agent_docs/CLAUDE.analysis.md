# Analysis Mode

실험 결과를 해석하고 의미를 도출하는 단계다. 확증 편향을 경계한다.

## Flow
1. 결과를 있는 그대로 기술 (해석 없이 사실만)
2. 가설과 결과가 일치하는가? 불일치하면 왜?
3. 대안적 설명을 최소 2개 생성
4. 통계적 유의성 / 재현성 확인
5. limitation을 명시적으로 나열

## Rules
- 결과가 가설과 맞더라도 "왜 틀릴 수 있는가?"를 반드시 검토
- subagent로 대안적 해석을 생성한다 (확증 편향 방지)
- 시각화를 먼저 만든다 — 숫자보다 패턴이 먼저 보인다
- limitation을 숨기지 않는다 — 정직한 limitation이 논문의 신뢰도를 높인다
- 결과 해석은 WORKLOG.md에 시간순 기록
