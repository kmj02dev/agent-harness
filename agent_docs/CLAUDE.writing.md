# Writing Mode

학술 논문/기술 문서를 집필하는 단계다. 명확성과 논리적 흐름이 핵심이다.

## Flow
1. 아웃라인 먼저 — 전체 구조를 잡고 시작
2. 핵심 기여(contribution)를 한 문장으로 정의
3. 각 섹션을 독립적으로 작성 (순서: Method → Results → Introduction → Discussion → Abstract)
4. subagent로 각 섹션 리뷰 (논리 비약, 근거 부족, 불명확한 표현)
5. 전체 흐름 검토 — 읽는 사람의 관점에서

## Rules
- 한 단락 = 한 아이디어. 단락이 두 가지를 말하면 분리
- 주장에는 반드시 근거를 붙인다 (데이터, 선행 연구, 논리적 추론)
- "we believe", "it is obvious" 같은 약한 표현 금지 — 증거로 말한다
- Related Work은 비교가 목적이다 — 나열이 아니라 "기존과 우리의 차이"를 명시
- Abstract은 마지막에 쓴다 — 전체를 다 쓴 후 정수만 추출
- 집필 중 새로운 실험이 필요하면 TODO.md에 추가하고 현재 집필을 계속한다 (모드 전환 최소화)
