# LESSONS

검증 전 가설·관찰을 보관한다. 검증되면 규칙으로 승격 후 제거. 반증되면 삭제.
**크기 관리**: 최대 20개. 항목마다 검증 조건 필수.

## Agent Design
- 35분+ 작업 시 모든 에이전트에서 성능 열화가 발생한다. 장기 작업은 체크포인팅 또는 세션 분리가 필수.
  **검증**: 3개 이상 프로젝트에서 세션 길이별 성공률 측정. 확인 시 CLAUDE.md로 승격.
- 멀티에이전트 4개 이상에서 조정 세금이 성능 이득을 초과한다. 2-3개가 최적.
  **검증**: 4+ agent 구성에서 실패 사례 2건 이상 수집. 확인 시 CLAUDE.md로 승격.
- 단순한 루프 + 좋은 도구 인터페이스가 복잡한 멀티에이전트보다 안정적이다 (Claude Code, SWE-agent 사례).
  **검증**: 동일 작업에서 두 접근법 비교 1회 이상. 확인 시 CLAUDE.md로 승격.

## Mode & Domain
- 유사 출처 ≠ 동일 출처. 원본 접근 불가 시 비슷한 제목의 다른 문서로 대체하면 잘못된 정보를 전달한다.
  **검증**: research 모드에서 재발 여부 관찰. 확인 시 CLAUDE.research.md로 승격.
- 도메인의 범작업적 지식(개념, 용어, 주요 문헌)이 없으면 도메인 내 판단(논문 동일성 등)을 할 수 없다.
  **검증**: 새 도메인 추가 시 base.md 없이 작업 → 실패 관찰. 확인 시 CLAUDE.domain.md로 승격.
